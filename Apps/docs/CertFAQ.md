# <a name="microsoft-365-certification-faq"></a>Microsoft 365 perguntas frequentes sobre certificação

## <a name="introduction"></a>Introdução
Abaixo, são frequentes perguntas frequentes (perguntas frequentes) feitas por ISVs (Fornecedores de Software Independentes) ao iniciar a certificação Microsoft 365.  Se houver alguma consulta que você tenha que não seja abordada aqui, entre em contato com a equipe Microsoft 365 de Certificação de Aplicativos por meio AppCert@Microsoft.com.
Este documento é voltado para ISVs, informações gerais sobre o programa Microsoft 365 Segurança e Conformidade podem ser encontradas na [página Microsoft 365 programa de Conformidade de Aplicativos](https://docs.microsoft.com/microsoft-365-app-certification/docs/enterprise-app-certification-guide). 


|Pergunta                                       |Definição expandida                          |
|----------------------------------------       |---------------------------------------------|
|Não passei em uma auditoria em uma estrutura reconhecida pelo setor, como PCI DSS, SOC 2 ou ISO 27001.  Isso significa que não posso solicitar a certificação Microsoft 365? | Não, obter uma dessas estruturas reconhecidas pelo setor não é um requisito da certificação Microsoft 365.|
|Já passei por uma auditoria externa em uma estrutura reconhecida pelo setor. Isso pode contar para a certificação Microsoft 365? | A resposta curta é Sim.  Atualmente, Microsoft 365 especificação de Certificação aceita evidências de estruturas externas PCI DSS, SOC 2 e ISO27001. O guia Microsoft 365 envios de certificação mapeou onde essas estruturas externas existentes se alinham; no entanto, descobrimos em algumas instâncias que o padrão/estrutura existente não se alinhe adequadamente. Por esse motivo, a equipe Microsoft 365 Certificação conduzirá uma revisão das evidências de padrões/estrutura fornecidas, marcando quais controles no Microsoft 365 Certificação são atendidos. |
|Como demonstrar a conformidade do RGPD se não fizemos uma avaliação externa do RGPD? | A Microsoft não exige uma revisão independente da conformidade do RGPD para a Certificação Microsoft 365, pois esse é um cenário em que aceitaremos autoteste que podemos verificar independentemente quando nenhuma revisão externa tiver sido realizada. Como isso é mais uma avaliação do que uma auditoria, e com relação às evidências a serem coletadas durante nossos processos, analisar políticas de privacidade e processos internos é como abordamos o controle RGPD. Para fins do que estamos procurando no controle RGPD, isso envolve principalmente a revisão de políticas de privacidade para garantir que elas atendem aos requisitos básicos do RGPD; por exemplo, quais dados pessoais estão sendo processados, qual é a legalidade do processamento, apontar os direitos do assunto de dados, como uma solicitação de acesso a assunto (SAR) será executada por um usuário, como o ISV realizará SARs, os detalhes da empresa ISVs e os detalhes de retenção de dados.|
|Fizemos um teste de penetração; no entanto, não temos um teste de penetração "limpo", pois não realizamos um novo teste de penetração.  Precisamos realizar um novo teste e ter um relatório limpo? | A especificação de certificação Microsoft 365 não exige isVs para realizar testes de penetração novamente, fornecendo evidências de correção adequadas podem ser fornecidas para demonstrar que os problemas identificados no relatório de teste de penetração são remediados.|
|Parte da documentação e das evidências solicitadas é confidencial, existem contratos de não divulgação (NDAs) em uso? | Sim, algumas das informações que você enviar serão informações públicas e algumas podem ser informações confidenciais.  Se você tiver um NDA existente com a Microsoft, os termos desse NDA serão aplicados às informações confidenciais que você enviar.  Se você não tiver um NDA com a Microsoft, os termos de confidencialidade do Contrato de Publisher assinado no Partner Center serão aplicados a essas informações confidenciais.|
|Como transferir documentação e evidências confidenciais com segurança como parte da avaliação Microsoft 365 Certificação? | Atualmente, a Microsoft não tem uma plataforma para compartilhar essas informações com segurança. A recomendação é que você compartilhe essas informações por meio de quaisquer mecanismos seguros que você já tenha. Muitos ISVs utilizarão OneDrive e compartilharão um link autenticado para a equipe Microsoft 365 Certificação.|
|Implementamos alguns processos de segurança adicionais para atender a alguns dos controles de Certificação Microsoft 365, isso significa que precisamos esperar 12 meses para podermos certificar?|Não, a Microsoft reconhece que talvez seja necessário desenvolver processos de segurança adicionais para fazer a ponte entre os processos de segurança existentes e o que é esperado da certificação Microsoft 365. A Microsoft 365 de Certificação analisará os processos documentados recentemente desenvolvidos e revisará as evidências de que o processo foi executado pelo menos uma vez. Evidências históricas adicionais não serão necessárias, pois isso não estará disponível para esses processos recém-desenvolvidos. Após doze meses, um exemplo de evidência histórica começará a ser avaliado durante a avaliação anual.|
|O que sou responsável por fornecer?| Durante a avaliação, os analistas de certificação revisarão documentos e evidências fornecidos para avaliar sua conformidade com os controles Microsoft 365 Certificação. Como parte deste trabalho, Microsoft 365 equipe de Certificação do Microsoft 365 solicitará informações que incluirão detalhes de arquitetura, diagramas, detalhes de armazenamento de dados, detalhes do design do aplicativo, documentos de política e processo, arquivos de configuração e capturas de tela. Em algumas ocasiões, ou se for mais fácil para você, uma sessão de compartilhamento de tela pode ser organizada para mostrar as evidências dos analistas de certificação. If, existing compliance frameworks are to be used to support the assessment activities, adequate documentation will be required demonstrating what the external auditor/assessor has assessed and confirmed as in place. Quando a documentação de suporte não puder fornecer a narração necessária para demonstrar exatamente como os controles dentro da estrutura de segurança externa foram atendidos, a equipe de Certificação do Microsoft 365 não poderá utilizar a estrutura de segurança externa para dar suporte à avaliação de certificação Microsoft 365.|
|Alcançar a certificação exigirá que eu faça alterações na minha infraestrutura atual?| É improvável que alterações significativas na infraestrutura sejam necessárias para atender à certificação Microsoft 365. Os controles são baseados nas práticas recomendadas de segurança do setor e provavelmente já serão implementados.  Vimos na maioria dos casos; Os ISVs tiveram que atualizar processos internos para fazer ponte entre as práticas de trabalho atuais e o que é necessário na Certificação Microsoft 365. Se isso for uma preocupação, a Microsoft recomenda que você revise os controles de Certificação Microsoft 365 mais recentes que podem ser encontrados no Guia de Envios de Certificação do Microsoft 365 para garantir que seu ambiente e as práticas de trabalho implantados no momento atendem aos controles definidos.|
|A Microsoft tem recomendações sobre componentes/infraestrutura/software específicos que devem ser usados para atender aos requisitos de certificação?| A Microsoft não fornece recomendações específicas sobre soluções para atender Microsoft 365 de Certificação. Qualquer oferta comercial ou de código aberto pode ser usada, desde que sejam ativamente suportadas e mantidas.|
|Quanto tempo leva para concluir a avaliação?| Normalmente, uma avaliação pode levar em média 30 dias para ser concluída, no entanto, isso pode depender de muitas variáveis. O tempo a ser concluído pode variar dependendo: o tamanho do ambiente de hospedagem usado para dar suporte ao Aplicativo/Complemento, o tipo de ambiente de hospedagem que dá suporte ao Aplicativo/Add-in e como os ISVs de aviso estão respondendo às solicitações de evidência.|
|Quanto tempo precisarei alocar para esse processo? | A maior parte do trabalho é simplesmente reunir a documentação e as evidências em tempo hábil. Depois disso, ele não deve exigir mais de algumas horas por semana para concluir o processo de avaliação. Algumas variáveis que podem afetar o tempo necessário são: o tamanho do ambiente terá um impacto sobre o tempo necessário para coletar evidências solicitadas e se houver alguma estrutura de segurança externa que possa ser aproveitada para dar suporte à avaliação.  Onde as estruturas de segurança externas estão em uso e a documentação de suporte adequada pode ser fornecida, os analistas de certificação podem usar essas avaliações externas para satisfazer um subconjunto de controles Microsoft 365, sem a necessidade de fornecer evidências adicionais.|
|Por que há um prazo fixo de 60 dias para a avaliação? | Definimos um limite no tempo em que uma avaliação pode ser realizada, pois as evidências já coletadas podem ficar mais demoradas quanto mais uma avaliação leva.  Esse é um ponto de avaliação de tempo e, portanto, precisa haver um período adequado alocado para conclusão. Depois de enviar seu envio inicial de documento, responderemos com uma solicitação de evidência. O período de 60 dias começa quando você recebe a solicitação de evidências. O [Microsoft 365 de Envios](https://docs.microsoft.com/microsoft-365-app-certification/docs/certification-submission-guide) de Certificação deve ser lido e você deve ter certeza de que todos os controles podem ser atendidos antes de enviar o envio inicial de evidências.|
|O que acontece se a avaliação não for concluída dentro do prazo de 60 dias? | Infelizmente, se a avaliação não for concluída durante o período de 60 dias, a Microsoft marcará uma falha na avaliação. Essa marca é apenas para estatísticas internas e nunca será publicada. Você poderá reiniciar imediatamente o processo de avaliação, no entanto, será solicitado a resendá-la para dar suporte ao novo aplicativo.|
|Quanto a certificação Microsoft 365 me custará? | Atualmente, é GRATUITO para você concluir a certificação Microsoft 365. |
|Qual é o custo do Teste de Penetração neste programa? | Se seu aplicativo deve passar por testes de penetração, onde isso não faz parte de suas atividades de segurança, os testes de penetração podem ser concluídos sob Microsoft 365 Certificação e é GRATUITO. O escopo do teste de penetração é limitado ao Aplicativo e à infraestrutura de suporte que está no escopo da certificação Microsoft 365. |
|Você tem materiais de marketing que podem ser usados para anunciar o fato de que nosso aplicativo foi certificado? | Atualmente, não oferecemos materiais promocionais ou de marketing. Embora nós o incentivemos a dizer aos clientes que seu aplicativo recebeu uma Certificação Microsoft 365, no entanto, os únicos elementos gráficos disponíveis serão aqueles mostrados na listagem de aplicativos no AppSource e nas páginas do Microsoft Docs. Atualizaremos este documento se algum material de marketing estiver disponível.
|Qual nível de evidência você está procurando ao executar a avaliação? | As evidências fornecidas durante Microsoft 365 avaliação de certificação devem ser capazes de fornecer garantia suficiente de que você está a atender aos controles específicos Microsoft 365 Certificação que estão sendo avaliados.  As evidências podem estar na forma de arquivos de configuração, capturas de tela de configurações ou evidências, documentação de política/procedimento ou sessões de compartilhamento de tela para demonstrar evidências para o analista de certificação.  A seguir estão dois exemplos:|
||**Atividade de** Avaliação: "Demonstre que o software antivírus está sendo executado em todos os componentes do sistema amostrados." – Para esse controle, você pode fornecer uma captura de tela de todos os dispositivos no exemplo que dão suporte a antivírus que mostra o processo anti-vírus em execução ou se você tiver um console de gerenciamento centralizado para antivírus, talvez seja capaz de demonstrar a partir desse console de gerenciamento.|
||**Atividade de Avaliação**: "Demonstrar como novas vulnerabilidades de segurança são identificadas". – Esse controle é da seção Gerenciamento de Patch. A intenção é que você tenha um processo formalmente documentado para identificar novas vulnerabilidades de segurança.  Isso pode estar no seu código-fonte, mas também precisa estar no ambiente de suporte, por exemplo, Windows vulnerabilidades, vulnerabilidades em dependências da Web (por exemplo, AngularJS, JQuery, etc.).  Você deve ter um processo documentado que você segue para identificar novas vulnerabilidades de segurança, portanto, deve fornecer o documento de processo documentado.  Além da documentação, você precisaria fornecer evidências de que o processo está sendo seguido; por exemplo, se você estiver utilizando algo como auditoria npm para verificar dependências de vulnerabilidades, fornecer uma amostra de relatórios fornecerá evidências.  Se você estiver utilizando vários processos, ou seja, para diferentes componentes do sistema, as evidências de todos os processos precisarão ser fornecidas.|
